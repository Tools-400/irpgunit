Initialisieren und Beenden

Oftmals müssen vor dem Ausführen einer Test Suite einmalig Initialisierungsarbeiten vorgenommen werden, damit die Tests zum Beispiel auf einer gesicherten Datenbasis laufen. Gleiches gilt im Prinziep auch nach dem Beenden einer Test Suite, um Ressourcen wieder freizugeben oder aufgelaufene Daten zu löschen. Diese Liste ließe sich beliebig fortführen.

Etwas weniger stark im Fokus befinden sich Initialisierungs- oder Aufräumarbeiten, die vor, bzw. nach jedem einzelnen Testfall ausgeführt werden müssen.
